## YOUR HUNTING GROUNDS

**1. SPECIFICATION ERRORS**

- Does the regression specification in the code match what the paper describes?
- Are fixed effects included/excluded as claimed?
- Are the control variables listed in the paper actually in the code?
- Does the sample restriction in the code match the stated sample?
- Are sampling weights, frequency weights, or analytical weights used as described — or silently added/omitted?
- Is the estimation method (OLS, IV/2SLS, probit, Poisson, etc.) what the paper claims?

**2. STANDARD ERROR SPECIFICATION**

Standard errors are frequently misreported. Check carefully:

- Are standard errors clustered at the level described in the paper?
- Is the clustering level consistent across specifications in the same table, or does it vary without explanation?
- Is the variance estimator (robust, clustered, bootstrapped, etc.) what the paper states?
- If clustering is applied, check the actual number of distinct clusters in the data — if it is very small (roughly below 30–50), flag this as the asymptotic approximation underlying cluster-robust standard errors may not be reliable, regardless of what the paper claims.

**3. ABSENCE ERRORS**

This is a common and easily missed class of error. Ask not only "is this wrong?" but "is this missing?"

- The paper describes an interaction term — is it actually in the regression?
- The paper says indicators are included as controls — are they in the code?
- The paper claims a restriction (e.g., consecutive years, balanced panel, specific subgroup) — is it enforced in the code?
- The paper describes a multi-step procedure — does the code implement every step, including steps that generate intermediate variables or apply the procedure to all relevant subgroups?
- For each result the paper presents, trace the corresponding code and ask: does the code produce exactly what is claimed, or does it silently omit a component?

**4. VARIABLE CONSTRUCTION vs. DESCRIPTION**

- For each key variable (outcome, treatment, instrument), find where it is constructed in the code.
- Does the construction match the verbal description in the paper exactly? Look for:
  - Wrong summary statistic (e.g., a mean when the paper describes a different aggregation)
  - Incorrect denominator or numerator
  - Wrong unit of observation (e.g., individual-level when group-level was described, or vice versa)
  - Wrong sample scope (e.g., a subset of observations when the full population was described)
  - Subsetting or conditioning that is not mentioned
- Pay particular attention to how zero or missing values in the denominator are handled in ratio variables — small-constant adjustments that affect sign or magnitude are a common undisclosed transformation.

**5. TREATMENT AND IDENTIFICATION VARIABLES**

For papers using difference-in-differences, instrumental variables, regression discontinuity, or event studies:

- Is the treatment indicator constructed exactly as described? Check: event date, treatment group criterion, aggregation level.
- Is the control/comparison group defined as the paper claims?
- Is the instrument aggregated at the correct level and using the correct variation?
- For event studies: is the event window defined consistently? Is the omitted (baseline) period the one stated in the paper?
- For RD designs: is the running variable constructed correctly? Is the bandwidth applied as described?

**6. SAMPLE PERIOD AND DATA VINTAGE**

- Does the code use the exact time window described in the paper?
- Are there off-by-one errors in year or period ranges (e.g., code runs 1991–2010 but paper states 1990–2010)?
- If the paper describes using a specific vintage or release of a dataset, does the code use that version, or a different one?

**7. UNDISCLOSED DATA MANIPULATION**

- Are there any filters, drops, or recodes that are not mentioned in the paper?
- Are any outcomes or variables winsorized, trimmed, or transformed in ways not disclosed?
- Are any observations excluded without justification?
- Is there evidence of undisclosed sample splits or specification searching?
- Are there multiple versions of a key variable in the code, with the analysis using one that differs from the described version?

**8. RESULT DISCREPANCIES**

- Do any hardcoded numbers in the code (elasticities, thresholds, cutoffs, reported statistics) match the numbers actually reported in the paper's tables? Check each one.
- Is there evidence that the code produces results different from those reported in the text?
- Are figures and tables constructed in a way that matches the paper's described layout?
- Where the paper presents results as deriving from a specific specification, confirm that the code generating those results matches that specification exactly.

**9. IDENTIFICATION ASSUMPTION TESTING**

Papers often *claim* to satisfy identification assumptions; the code may not actually test them.

- For **difference-in-differences**: does the paper claim parallel pre-trends? Does the code test this — i.e., does it run an event study or pre-trend test and report the results? If the paper claims pre-trends are tested but the code does not include this test, flag it.
- For **instrumental variables**: is the first-stage F-statistic reported in the paper? Does the code actually compute and output it? A weak instrument (conventionally F < 10, though modern thresholds are higher) invalidates the IV approach. Does the paper defend the exclusion restriction beyond assertion?
- For **regression discontinuity**: does the code implement a density test (McCrary test or equivalent) to check for sorting at the cutoff? Does it test for discontinuities in pre-determined covariates at the threshold? If the paper claims these checks were done, is the code present?
- For **event studies**: are the pre-period coefficients jointly tested for significance? Is this test reported? Does the paper claim coefficients are near zero pre-event — and does the code actually test this or only present a figure?

**10. MULTIPLE HYPOTHESIS TESTING**

Selective reporting across many outcomes or subgroups is a common source of inflated results.

- Does the paper test effects on multiple outcome variables, in multiple subgroups, or across many specifications? Count how many distinct hypothesis tests are effectively being run.
- Are p-values or confidence intervals adjusted for multiple comparisons (e.g., Bonferroni, Benjamini-Hochberg, Romano-Wolf)? If the paper tests many outcomes without correction, flag this.
- Does the paper report only a subset of outcomes or subgroups tested? Look for signs in the code that more specifications were run than are reported — variables constructed but not used in final tables, commented-out regressions, or loops over outcomes where only some are presented.
- Is there evidence of **specification searching** — multiple versions of the same regression with slightly different controls, samples, or variable definitions, where only the significant version is reported?

**11. UNIMPLEMENTED CLAIMS**

- Are there robustness checks claimed in the paper that have no corresponding code?
- Are there figures or tables in the paper with no obvious code to generate them?
- Does the paper claim a method was applied that has no trace in the replication package?
- Are there placebo tests, falsification checks, or sensitivity analyses described but not coded?
