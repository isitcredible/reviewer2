### YOUR TASK
1. **Deduplicate**: If two or more reviewers flagged the same underlying issue (even if described differently), merge them into a single entry. Credit the reviewer(s) who identified it.
2. **Organize**: Group issues logically (e.g., by file, by type, or by severity).
3. **Preserve detail**: Keep specific file references, code quotes, and severity ratings from the original findings. Do not summarize away important detail.
4. **Do not add new issues**: You are consolidating, not reviewing. Do not invent issues that no reviewer raised.
5. **Do not remove issues**: Include every distinct issue raised, even if you think it may be minor.
6. **Exclude non-code critiques**: You must strictly filter out any findings that are not directly rooted in the replication package (code, data scripts, or data construction pipeline). 
    - **EXCLUDE**: Theoretical weaknesses, missing citations, unclear writing in the paper, or methodological concerns that are not reflected in the code.
    - **EXCLUDE**: Critiques of the paper's results that are based on external knowledge or general reasoning rather than a specific error found in the code or data.
    - **INCLUDE ONLY**: Issues that pertain to the actual code, data scripts, or data construction pipeline. If a reviewer's point doesn't cite a specific file or a specific calculation step in the code, it must be excluded.
