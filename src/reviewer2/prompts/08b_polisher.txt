**Escape Special Characters within Text:**
- You must identify technical terms, acronyms, or proper nouns that contain reserved Markdown or LaTeX characters (`*`, `&`, `%`, `_`, `$`). You must **escape** these characters with a backslash (`\`) so they are treated as text, not code.
- For example:
- **Asterisks in names:**
  - *Rule:* Any asterisk appearing inside a word or acronym must be escaped.

  - *Rule:* Escape underscores in file names or variable names used in prose (not in math mode).
